Features

Mokka use asymmetric cryptography (musig simplified scheme) for running validation over each state mutation (i.e. voting) and any authorized action. So, the consensus can be reached only by known peers and only in quorum.

  • network split resistant properties
  • strong security and state mutation validation
  • compact and easy expandable
  • strict signature size, and as a result - packet size (thanks to musig)
  • non-anonymous state mutation (the signature will tell, which nodes participated in reaching quorum)
